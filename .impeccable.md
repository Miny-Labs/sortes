# Sortes — Design Context

Synthesized from the conversation that produced the v4 frontend and the
`taste-skill` design discipline that was applied. Update directly if any
of this becomes wrong.

## Design Context

### Users
Crypto-native traders using a prediction market. They land on the home page
expecting to scan many markets quickly, click into one, and place a bet
against on-chain odds. Some of them care specifically about privacy
(whales who don't want to telegraph direction or stake size). The UI is used
on a desktop monitor most of the time, sometimes a tablet, occasionally
mobile when checking positions.

### Brand Personality
Sortes — sealed-bid prediction markets. Three words: **refined, technical,
honest**. Not crypto-bro hype. Not corporate fintech sterility. Closer to a
trader's terminal that respects the user. The interface should *feel*
sealed: hairline borders, restrained palette, monospaced numbers, only
enough light/motion to convey state.

### Aesthetic Direction
- **Theme**: dark. Audience is doing focused, fast trading work with
  numbers and probabilities; dark mode reduces eye strain and matches the
  trader-terminal mental model.
- **Type**: Geist + Geist Mono (chosen during the taste-skill rebuild).
  Mono for all numbers/addresses/timers — never for body copy.
- **Color**: zinc/ink neutrals + a single emerald signal accent for YES /
  positive states + rose for NO / negative. No purple-blue gradient. No
  neon glow. No gradient text.
- **Layout**: asymmetric hero (split column), bento markets grid with
  varied card sizes, right-side drawers for market detail and wallet —
  not separate pages. Two routes total.
- **Anti-references**: generic Polymarket clone, AI-purple SaaS dashboards,
  glassmorphism-everywhere, hero metric cards with sparkline decorations,
  emoji-as-icons.

### Design Principles
1. **Sealed > shiny.** Hairlines, not glow. Rare accent, not gradient soup.
2. **Mono for numbers.** Every USDC.e amount, address, percentage, timer
   uses Geist Mono with tabular-nums. Never serif numerals.
3. **Aggregate, not exposed.** When data is encrypted (sealed direction,
   pre-reveal odds), the UI says so explicitly with the Lock icon and a
   one-liner — never with mystery dashes.
4. **Two routes, drawers for everything else.** Resist the urge to add
   nav. New surface area is a drawer, not a page.
5. **No AI tells.** No 99.99%, no "Acme/Nexus", no Inter, no purple
   gradients, no #000000, no 3-equal-card grids, no border-left stripes.
